Unified Modeling Language (UML)

UML adalah bahasa pemodelan standar untuk menjelaskan, memvisualisasikan, merancang, dan mendokumentasikan sistem. Dengan UML kita dapat mempresentasikan struktur, perilaku, dan arsitektur fisik sistem menggunakan berbagai diagram. (Ringkasan materi diambil dari modul yang kamu upload).

🎯 Use Case Diagram

Use case menggambarkan fungsi sistem dari perspektif pengguna (aktor). Use case merepresentasikan scenario utama di mana aktor berinteraksi dengan sistem — misalnya "Lihat Denah", "Bayar SPP", "Cek Jadwal". Komponen penting: aktor, use case, dan system boundary. Use case berguna untuk menyusun requirement dan komunikasi rancangan dengan klien.

Contoh penggunaan: dalam SmartSchool, aktor = Siswa/Guru/Admin, use case = "Lihat Lantai", "Top-up Bank Mini", "Lihat UML Module".

⚙️ Activity Diagram

Activity diagram menggambarkan alur kerja (flow) dari suatu use case, mirip flowchart tetapi mendukung perilaku paralel. Simbol penting: initial node, action/state, decision (diamond), fork/join (paralel), dan final node. Activity diagram berguna saat menjelaskan detil proses bisnis.

Contoh: proses "Bayar SPP" — siswa memilih metode, melakukan transfer, konfirmasi, dan update saldo.

📶 Sequence Diagram

Sequence diagram menggambarkan interaksi antar objek/participant menurut waktu (atas → bawah). Garis vertikal = lifeline; kotak activation menunjukkan waktu eksekusi; panah mewakili message. Diagram ini berguna untuk menjelaskan alur pesan dalam sebuah scenario (use case).

Contoh: Siswa → Web → Server → Bank (untuk proses top-up) — setiap langkah muncul sebagai panah berurutan.

🏛️ Class Diagram

Class diagram menampilkan struktur statis sistem: kelas, atribut, operasi, dan relasi (association, aggregation, inheritance, realization). Class memiliki tiga area pokok: nama, atribut, dan metoda. Class diagram berguna untuk mendesain struktur data dan API internal.

Contoh kelas pada SmartSchool: User, Ruangan, Lantai, TransaksiBank.

💾 Deployment / Development Diagram

Deployment diagram menggambarkan arsitektur fisik: node (server, client), artefak, dan bagaimana komponen ditempatkan di lingkungan runtime. Diagram ini penting untuk perencanaan infrastruktur dan instalasi sistem.

Contoh: web server (hosting HTML/CSS), database server, dan client browser yang mengakses halaman denah.

Langkah-langkah Penggunaan UML (Ringkasan)

1. Buat daftar business process level tertinggi.
2. Petakan use case untuk tiap business process dan lengkapi requirement.
3. Buat deployment diagram kasar untuk arsitektur fisik.
4. Definisikan requirement non-fungsional (security, performance).
5. Dari use case → buat activity diagram.
6. Identifikasi objek domain → buat sequence / collaboration diagram.
7. Rancang UI model untuk skenario use case.
8. Buat class diagram dan unit test untuk tiap class.
9. Kelompokkan class menjadi komponen → buat component diagram.
10. Perhalus deployment diagram (mapping komponen → node).
11. Bangun sistem (use case-driven atau component-driven).
12. Lakukan unit & integrasi test; sinkronkan model dan code.
13. Rilis perangkat lunak.
